# AI-Listener

マイクで拾った配信者の声を Gemini AI が解析し、Twitch 常連リスナーとして自動コメントを生成する OBS 連携ツールです。  
**静的ファイル1枚 + HTTPS ホスティング** で動作します。サーバー不要・インストール不要。

---

## 動作の仕組み

```
マイク音声 → Web Speech API（ブラウザ内蔵STT）→ 辞書補正 → Gemini API → チャットバブル表示
```

- HTTPS で配信されるため OBS の起動フラグ変更は**不要**
- APIキーはブラウザの localStorage にのみ保存（ソースコードには含まれない）

---

## 初回セットアップ（一度だけ）

### 1. Gemini APIキーを取得

[Google AI Studio](https://aistudio.google.com/) でAPIキーを発行します（無料枠あり）。

---

### 2. ファイルを HTTPS でホスティング

`controller.html` を以下いずれかにデプロイしてください。

#### Cloudflare Pages（推奨・最も簡単）
1. [Cloudflare Pages](https://pages.cloudflare.com/) にサインイン
2. 「新しいプロジェクトを作成」→「直接アップロード」を選択
3. `controller.html` をドラッグ＆ドロップ
4. 発行された `https://xxxxx.pages.dev` の URL をメモ

#### GitHub Pages
1. GitHub に `ai-listener` リポジトリを作成
2. `controller.html` を追加してプッシュ
3. Settings → Pages → Source を `main` ブランチに設定
4. `https://ユーザー名.github.io/ai-listener/controller.html` の URL をメモ

---

### 3. OBS にカスタムブラウザドックを追加

1. OBS → **ツール → カスタムブラウザドック**
2. 「追加」をクリック
3. 名前: `AI-Listener`、URL: 上記でメモした HTTPS URL を入力
4. 「閉じる」でドックが表示される
5. マイク使用の許可ダイアログが出たら **「許可」** をクリック

---

### 4. APIキーを入力して開始

1. ドック内のセットアップ画面で **「APIキー」** 欄にキーを貼り付け
2. **「保存して開始」** をクリック
3. チャット画面に切り替わり、ステータスが **「● 待機中...」** になれば完了

---

## 日常の使い方

- OBS を起動するだけで自動的に待機状態になります（APIキーは保存済み）
- マイクに向かって話すと自動で認識 → AI が返答バブルを表示します
- 右上の **⚙ ボタン** で設定画面に戻れます（APIキー変更・辞書編集）

| ステータス | 意味 |
|---|---|
| `● 待機中...` | 発話待ち |
| `◉ 認識: ...` | 発話を検出・テキスト化中 |
| `◎ 考え中...` | Gemini に返答を問い合わせ中 |
| `⚠ レート制限 — 1分待機` | API 制限に当たった（自動回復） |

---

## 辞書機能（誤認識フィルター）

Web Speech API の誤変換を Gemini 送信前に自動補正します。

**設定画面 → 辞書セクション** から追加できます。

| 誤認識ワード | 置き換え後 |
|---|---|
| `リアルホース` | `REALFORCE` |
| `しずく` | `Sizzle` |

- 辞書データはブラウザの localStorage に保存されます
- 削除ボタンで個別に削除できます

## CONFIG 調整（上級）

`controller.html` 内の `CONFIG` オブジェクトを編集することで動作を調整できます。

| パラメータ | 初期値 | 説明 |
|---|---|---|
| `GEMINI_MODEL` | `"gemini-2.5-flash"` | 使用するモデル |
| `TIMEOUT_MS` | `3000` | API タイムアウト（ms） |
| `COOLDOWN_MS` | `60000` | レート制限後の待機時間（ms） |
| `HISTORY_MAX` | `5` | 会話履歴の保持ターン数 |
| `MAX_BUBBLES` | `20` | 画面上の最大バブル数 |
| `TYPEWRITER_MS` | `30` | タイピングアニメーション速度（ms/文字） |
| `MIN_CHARS` | `3` | これ以下の文字数の発話は無視（ノイズ除去） |

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| `マイクへのアクセスを許可して…` と表示 | OBS のドックでマイク許可ダイアログを「許可」に変更 |
| `Web Speech API に対応していません` | OBS または Chrome/Edge の最新版を使用してください |
| `⚠ APIエラー 400` | APIキーが正しくない可能性があります |
| `⚠ APIエラー 429` | レート制限。1分後に自動回復します |
| `⚠ タイムアウト` | ネット接続を確認。`TIMEOUT_MS` を増やすと改善する場合があります |
| 発話しても反応しない | `MIN_CHARS` を下げる、またはマイクのレベルを上げる |

## OBS ブラウザソースとして使う場合

カスタムブラウザドックではなく **シーンのオーバーレイ** として使いたい場合：

1. OBS → ソース追加 → **「ブラウザ」**
2. URL に HTTPS URL を入力
3. **「ページ背景を透明にする」** にチェック（チャット画面の背景が透過されます）
4. 幅・高さをシーンに合わせて設定
